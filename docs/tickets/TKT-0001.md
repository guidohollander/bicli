# TKT-0001 Rule-Based BIXML Syntax Validator

## Request

Build a simple CLI that can validate the syntactical correctness of one or more Be Informed `.bixml` files, using information available in `C:\bi\Be Informed AMS 23.2.9`, especially the shipped JAR files.

## Constraints

- Prefer offline validation.
- Prefer not to run Be Informed.
- Start with one or more local `.bixml` files.
- Keep the delta small and safe.

## Initial interpretation

`TKT-0001` targets syntax validation, not full Studio-equivalent consistency checking. The first implementation derives rules from:

- `plugin.xml` namespace registrations
- `resources/*-mapping.xml` Castor mappings in the shipped plugin JARs
- optional project-specific custom plugin JARs when a project root is provided

## Acceptance

- CLI accepts one or more file paths.
- CLI reports pass/fail per file.
- CLI flags malformed XML.
- CLI flags unknown namespaces.
- CLI flags unknown or structurally unsupported element or attribute names based on shipped mappings.
