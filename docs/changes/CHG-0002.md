# CHG-0002 Versioned Be Informed Representation

## Summary

Added a canonical internal representation for Be Informed repositories and models in:

- `docs/canon/be-informed-representation.md`

## Why

`bicli` needs a stable model of:

- Be Informed version profiles
- project layering
- artifact categories
- artifact links

to move beyond syntax validation toward validation, debugging, creation, and maintenance support.

## Notes

- The representation is version-aware.
- It is based on non-frontend repositories under `C:\repo`.
- It treats `23.2.6` as the dominant current baseline and `24.2.6` as the newer compatible family that should still be modeled separately.
